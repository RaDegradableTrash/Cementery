当前碰撞逻辑说明（简化版）

一、总体设计目标
1. 规则尽量少，避免多套碰撞修正互相打架。
2. 优先稳定性，不追求复杂的物理反馈。
3. 重点防止两类问题：异常撞飞、穿模/抖动。

二、玩家与可推动物体（PlayerController）
1. 玩家撞到可推动物体时，不再使用冲量（AddForceAtPosition）。
2. 现在只改物体的“水平速度”，并且按质量衰减推动速度：
   - pushSpeed = pushForce / max(1, mass)
3. 推动速度是渐进逼近（MoveTowards），而不是瞬时爆发。
4. 保留物体原有竖直速度，不给额外上抬或下压分量。
5. 每次推动后清零角速度，避免因为接触点偏移导致乱转和弹飞。

三、携带物体（InteractionSystem）
1. 被携带物体始终朝“目标持握点”平滑插值移动与旋转。
2. 目标持握点采用简单约束：
   - 方向保持水平（不因低头把物体拖进脚边）。
   - 通过一次 SphereCastAll 做前向距离夹取，前方有障碍就提前停下。
   - 保证最小离地高度，避免贴地抖动。
3. 删除了复杂的逐次穿透修正流程（ComputePenetration 循环）。
4. 删除了“携带物体去推开其它物体”的额外冲量逻辑。
5. 携带期间仍同步刚体姿态并清零线速度/角速度，防止物理残留反推。

四、拿起/放下（InteractionSystem）
1. 拿起时：
   - 缓存原刚体状态（isKinematic、useGravity、interpolation、collisionDetectionMode）。
   - 设为 isKinematic = true，useGravity = false。
   - collisionDetectionMode 切到 ContinuousSpeculative，降低穿透概率。
2. 携带期间：
   - 忽略玩家与被携带物体的碰撞，防止玩家自身把物体顶偏。
3. 放下时：
   - 还原拿起前缓存的刚体状态。
   - 清零速度，避免放手瞬间继承异常速度。

五、世界物体可推动性（WorldObject）
1. 规则改为二分：
   - canBePushed = true：刚体设为动态（isKinematic = false）。
   - canBePushed = false：刚体强制运动学，并清零速度。
2. 去掉了每帧 Update 的重复强制，仅在 FixedUpdate 统一维护，减少抖动来源。
3. 物体被携带或在缩放动画期间，不在这里抢控制权，避免状态冲突。

六、当前行为总结
1. 玩家推物体：缓推、可控、低风险，不易把物体打飞。
2. 携带物体：沿中心轴稳定跟随，遇障碍自动收距，减少穿墙。
3. 不可推动物体：稳定锁定，不会被意外外力带走。
4. 整体从“复杂修正链”改为“少规则、明确优先级”，异常概率显著下降。
